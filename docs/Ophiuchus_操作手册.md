# Ophiuchus 操作手册

适用版本：v0.2.0-dev

维护与反馈：wanyc@issp.u-tokyo.ac.jp

## 1. 软件用途

Ophiuchus 是一套本地优先的材料科研工作台，目前重点用于：

- 导入实验 XRD 数据；
- 从本地 CIF、峰表或结构库中筛选候选相；
- 将实验谱、目标相模拟谱和可能杂质放在同一张图中比较；
- 检查候选相缺失强峰、实验未解释峰和实际参与分析的结构；
- 对候选相进行手动物相剥离、残差谱检查和受约束 RIETAN 精修；
- 从 Materials Project 采集公开结构，并跳过重复记录。

Ophiuchus 的候选分数和置信标签不是“物相存在概率”。它们用于排序证据、缩小排查范围，不能代替人工晶体学判断、完整 Rietveld 精修或授权数据库检索。

## 2. 运行条件

### 2.1 普通用户：便携版

收到 `Ophiuchus_Portable.zip` 后，先完整解压，再双击文件夹最外层的 `Ophiuchus.exe`。不要只把 exe 单独拖出来，也不要在压缩包预览窗口内直接运行。便携版已经包含锁定版本的 Python、NumPy、SciPy、Matplotlib、pymatgen 和 mp-api，接收者不需要安装 Anaconda、Python 或这些库。

软件会把个人配置、API Key、本地结构库、缓存和用户主动保存的结果写入 `%LOCALAPPDATA%\Ophiuchus`。替换或删除便携版程序文件夹不会自动删除这些个人数据。VESTA、RIETAN-FP 和 `cif2ins` 受各自许可约束，不随便携包分发；未配置时，相关按钮会显示不可用或明确的降级状态，不影响核心候选筛选和 pymatgen 模拟。

### 2.2 源码开发：推荐环境

- Windows 10 或 Windows 11，64 位；
- Anaconda 或 Miniconda；
- Python 3.11；
- 能够正常显示中文的系统字体；
- 如需在线采集：可访问 Materials Project，并使用自己的 API Key；
- 如需 VESTA/RIETAN 展示谱或受约束精修：本机安装或配置相应程序。

### 2.3 首次安装 Python 环境

在 Anaconda Prompt 中进入 Ophiuchus 文件夹，然后执行：

```powershell
conda create -n ophi python=3.11 -y
conda activate ophi
python -m pip install -r requirements.txt
```

主要依赖包括 `numpy`、`pandas`、`scipy`、`matplotlib`、`pymatgen` 和 `mp-api`。CIF 派生的模拟 XRD 必须使用经过验证的计算后端；后端失败时，Ophiuchus 会报告失败，不会悄悄换成未经验证的简化计算。

## 3. 启动软件

### 3.1 双击启动

便携版用户双击解压目录最外层的 `Ophiuchus.exe`。源码开发目录也可双击项目根目录中的 `Ophiuchus.exe`，在不显示命令行黑窗的情况下启动；也可以双击 `start_ophiuchus.bat`，启动脚本会优先寻找已配置的 `ophi` 环境，找不到时再尝试系统 Python。修改启动器源码后，双击 `build_launcher_exe.bat` 可重新生成 exe。

项目文件夹可以放在任意本地目录。需要桌面入口时，双击 `install_desktop_shortcut.bat`；它会根据当前项目位置和当前用户的真实桌面自动创建 `Start Ophiuchus` 快捷方式。移动项目文件夹后，再运行一次该安装脚本即可更新桌面入口。

### 3.2 命令行启动

在项目根目录执行：

```powershell
conda activate ophi
python -m ophiuchus app
```

如果窗口没有出现，先在同一终端运行上述命令并阅读错误信息，不要连续重复双击。

## 4. 主界面说明

主界面分为三部分：

1. 左侧：工作区导航和“帮助与关于”；
2. 中间：本次分析的输入、元素范围、目标相和主要操作；
3. 右侧：运行状态、XRD 图、候选相、结构库、实际使用结构和 Peak Inspector。

常用按钮含义：

| 按钮 | 用途 |
| --- | --- |
| 开始候选筛选 | 直接扫描指定候选文件夹并进行快速候选排序 |
| 连接 API / 数据库 | 保存并测试 Materials Project 配置，按当前元素采集结构 |
| VESTA / RIETAN 设置 | 设置程序路径、参考峰目录并执行预检 |
| 构建/更新缓存 | 预先模拟候选文件夹中的结构，减少重复计算 |
| 导入结构库 | 把候选文件夹中的本地 CIF 导入 SQLite 结构库 |
| 结构库分析 | 使用结构库中启用且符合主元素范围的结构分析实验谱 |
| 保存本次分析 | 把当前临时分析明确保存到用户选择的位置 |
| 保存图片 | 仅保存右侧当前 XRD 图 |
| 手动物相剥离 / 残差谱分析 | 检查各候选相如何共同解释实验谱 |
| RIETAN 受约束精修 | 在满足前置条件后启动受约束精修工作流 |

## 5. 第一次使用前的配置

### 5.1 选择实验 XRD

点击实验谱路径旁的选择按钮。支持 Rigaku 风格 `.asc` 以及常见的两列 `2theta/intensity` 文本文件。

导入后检查：

- 角度范围是否合理；
- 强度列是否正确；
- 文件名中的化学式是否与实际样品一致；
- 右侧预览是否存在明显错列、空谱或异常截断。

### 5.2 设定主元素范围

点击“元素周期表”，选择样品确认存在的主要元素。结构库分析只检查与主元素范围相关的结构，不会把所有元素全部试一遍。

主元素不是杂质猜测列表。氧化物等常见背景候选可通过数据库窗口中的氧化物补充功能加入结构库，但最终仍需根据实验条件判断是否合理。

### 5.3 设置候选文件夹和结构库

- 候选文件夹：保存本地 CIF、VESTA `.int` 或其他峰表；
- 本地候选缓存：快速候选筛选使用的 SQLite 缓存；
- 本地结构库：结构库分析使用的 SQLite 数据库。

首次使用可保留默认位置。共享给其他用户后，应在对方电脑上重新选择这些路径，不要继续使用原作者电脑的绝对路径。

## 6. 导入本地 CIF

### 6.1 批量导入结构库

1. 选择包含 CIF 的候选文件夹；
2. 确认本地结构库路径；
3. 点击“导入结构库”；
4. 等待状态栏显示完成；
5. 在右侧“结构库”标签页点击刷新，检查结构名称、来源、启用状态和 XRD 缓存状态。

结构库使用结构哈希和来源标识减少重复记录。导入失败时应查看日志中的具体 CIF 路径，而不是假定所有文件都已成功。

### 6.2 单独导入目标相

1. 先选择结构库数据库；
2. 点击“导入目标相 CIF”；
3. 选择目标相 CIF；
4. 点击“刷新目标相”；
5. 在下拉框中明确选择目标相。

改变主元素范围后，Ophiuchus 会要求重新确认目标相，以避免沿用上一次样品的错误目标。

## 7. 连接 Materials Project

1. 点击“连接 API / 数据库”；
2. 数据库选择 `Materials Project`；
3. 粘贴你自己的 Materials Project API Key；
4. 保存配置并测试连接；
5. 回到主界面确认主元素范围；
6. 在数据库窗口选择采集模式；
7. 点击按当前元素采集；
8. 完成后刷新右侧结构库。

建议第一次使用保守模式，确认目标相和主要二元、三元候选已进入结构库后再扩大范围。采集会按 Materials Project 标识和本地结构记录跳过重复结构。

API Key 保存在本机项目目录的 `.env` 中。每位用户必须使用自己的 Key；不要把 `.env` 放进共享压缩包、邮件附件或公共仓库。

COD、AFLOW、OQMD 和 NOMAD/OPTIMADE 等入口可能显示在界面中，但未接入的提供方会明确提示“当前版本尚未接入”，不会伪装成成功采集。

## 8. 工作流 A：快速本地候选筛选

适合已经有一个较小候选文件夹、希望快速检查哪些相值得优先看的情况。

1. 选择实验 XRD；
2. 选择候选结构或模拟峰文件夹；
3. 用元素周期表确认主元素；
4. 必要时先点击“构建/更新缓存”；
5. 点击“开始候选筛选”；
6. 在右侧查看候选相、缺失强峰、未解释峰和 XRD 图；
7. 确认结果有价值后，再点击“保存本次分析”或“保存图片”。

此流程直接扫描候选文件夹。它与结构库分析的候选来源不同，因此比较两次结果时必须确认使用的是同一组 CIF/峰表、同一元素范围和同一目标相。

## 9. 工作流 B：结构库分析

这是推荐的可追溯分析流程。

1. 选择实验 XRD；
2. 设定主元素；
3. 选择本地结构库；
4. 导入本地 CIF 或从 Materials Project 采集；
5. 在“结构库”标签页刷新并确认结构已启用、缓存可用；
6. 点击“刷新目标相”；
7. 在目标相下拉框中明确选择目标相；
8. 点击“结构库分析”；
9. 观察预计分析结构数和运行日志；
10. 在右侧核对候选排名、实际使用结构、跳过原因和 XRD 图。

结构库分析会使用主元素范围内的相关结构。目标相用于解释已知主相；其他候选优先解释目标相没有覆盖、但实验谱中真实出现的峰。

## 10. 如何阅读 XRD 图和候选结果

右侧展示图通常包括：

- 实验 XRD 曲线；
- 用户明确选择的目标相模拟谱或参考谱；
- 最多三个可能杂质候选；
- 与候选理论峰对应的竖向虚线；
- 必要的目标相峰位标注。

判断候选时至少同时检查：

1. 峰位是否在容差范围内对应；
2. 候选的强理论峰是否在实验谱中缺失；
3. 实验中目标相未覆盖的峰能否由候选解释；
4. 高角区峰是否系统偏移；
5. 模拟相对强度是否受织构、仪器、波长、占位、热振动或展宽模型影响；
6. 候选组成是否符合样品元素、合成条件和可能氧化过程。

候选分数高只表示证据排序靠前。仅凭一两个重合峰不能确认物相；明显缺失的强理论峰通常比偶然重合峰更值得警惕。

## 11. Peak Inspector 与结构可追溯性

在右侧 `Peak Inspector` 输入一个实验 `2theta`，程序会列出结构库缓存中附近的理论峰，包括结构标识、CIF 路径、hkl、强度和启用状态。

右侧“实际使用结构”用于回答：

- 哪些结构真正进入了本次评分；
- 哪些结构被跳过；
- 跳过原因是元素范围、禁用、缓存失败还是结构问题；
- 当前结果对应哪一个本地 CIF。

遇到可疑候选时，应先从这里追溯到原始 CIF，再用 VESTA 或其他可靠工具独立检查。

## 12. 手动物相剥离与残差谱

完成一次有效分析后，点击“手动物相剥离 / 残差谱分析”。推荐顺序：

1. 先预览目标相贡献；
2. 检查重建谱、残差谱和峰级解释；
3. 接受可信相；
4. 再预览下一个候选；
5. 使用撤销、重做或重置检查组合是否稳定；
6. 只有在需要时才显式导出 CSV、JSON 或 PNG。

残差会受到基线、峰宽、零点偏移、织构和强度缩放影响。不要把“残差下降”自动解释为“物相已确认”。

## 13. VESTA / RIETAN 与受约束精修

### 13.1 配置与预检

点击“VESTA / RIETAN 设置”，配置：

- `VESTA.exe` 路径；
- RIETAN-FP 可执行文件路径；
- 本地 VESTA 参考峰目录；
- 必要时为特定化学式指定参考文件。

设置窗口分别显示 RIETAN 单相模拟和多相实验定量是否就绪。若显示“多相实验定量：缺少官方模板”，点击“安装多相支持”。软件会优先寻找本机已有的官方 `Windows_versions.zip`；未找到时会让用户选择该压缩包。安装器只提取 RIETAN 官方 `Cu3Fe4P6_combins/template.ins`，并检查 `Elements / Phase / Parameters / Constraints @N` 和 `NPHASE@` 标记，不接受任意 `.ins` 文件冒充多相模板。官方程序、许可和引用说明见 <https://jp-minerals.org/rietan/>。

该按钮主要用于管理路径和检查可用性。正式候选筛选或结构库分析会在需要时调用已配置后端生成最终展示谱。

### 13.2 受约束精修

“RIETAN 受约束精修”只有在本次分析具有可用目标候选和 CIF 来源时才会启用。精修结果必须检查收敛情况、参数边界、残差结构和物理合理性，不能只看一个总残差值。

精修窗口包含两种模式：

1. `单相确认`：沿用原有受约束流程，用于检查一个目标相能否解释实验谱，不输出物相质量分数；
2. `多相实验定量`：明确选择一个目标相和一至三个杂质相，运行三组不同初始尺度的真实 RIETAN-FP 联合精修。

多相模式首版固定以下边界：Cu Kα1+2；原子坐标、占位、热参数、择优取向和各向异性展宽保持锁定；只允许 CIF 支持的 2 至 4 个互不重复结构。软件直接绘制 RIETAN 的实验谱、总计算谱、背景、差值曲线和各相反射标记，不把反射棒重新展宽成伪造的“各相贡献曲线”。

质量分数来自 RIETAN 最终尺度因子，并用 CIF 的 `Z × M × V` 做 Hill-Howard 交叉核对。候选分数、峰面积和残差信号占比不会被当作 wt%。等比例、目标相占优和杂质相占优三组起点必须收敛到稳定结果。

出现以下任一情况时，界面会显示“不可定量”并隐藏精确 wt%：

- 某相尺度因子、Z、摩尔质量或晶胞体积无效；
- 某个候选没有通过独立模拟谱验证；
- RIETAN 原生质量分数与 ZMV 交叉核对不一致；
- 差值曲线仍有成组强正峰，说明物相模型不完整；
- 三组初始尺度结果不稳定或物相顺序不一致。

通过硬门槛的结果仍标记为“实验性定量”。在使用已知组成的实测标样验证整套仪器、制样和精修流程前，不应把该数字用于正式定量结论。非晶成分、未加入模型的物相、严重择优取向、微吸收和不正确的背景都会造成系统误差。

临时 RIETAN 文件随任务结束删除。只有点击精修窗口中的导出按钮才会保存 PNG、总谱 CSV、各相反射 CSV 和带 CIF 哈希、引擎路径、参数哈希、可信度结论及稳定性范围的 JSON。

若直接模拟失败，Ophiuchus 会保留并标明筛选后端结果或失败状态，不会把其他计算冒充为 VESTA/RIETAN 输出。

## 14. 保存策略

桌面分析默认保存在临时会话中，只保留当前一次分析。这样反复试参数不会不断堆积完整结果文件。

- 点击“保存本次分析”：选择永久目录并保存完整结果；
- 点击“保存图片”：只保存当前右侧 XRD 图；
- 残差谱和精修窗口：分别使用各自的显式导出按钮；
- 命令行分析：仍会直接写入指定的输出目录。

关闭软件前，如果当前结果需要保留，必须主动保存。

## 15. 常见输出文件

根据分析路径不同，保存目录可能包含：

- `results.json`：完整机器可读结果；
- `report.md`：面向人工检查的报告；
- `top_candidates.csv`：候选相排序；
- `peak_assignments.csv`：实验峰与候选峰对应；
- `candidate_usage_summary.csv`：所有结构的使用/跳过证据；
- `xrd_presentation.png`：汇报风格 XRD 图；
- `xrd_diagnostic.png`：峰匹配和缺失峰诊断图；
- `audit_*/peak_match_table.csv`：更详细的峰级证据。

## 16. 共享给其他用户前

建议从一份干净副本制作共享包。不要直接把正在使用的整个工作目录原样压缩。

共享包应包含：

- `ophiuchus/` 源码；
- `docs/` 文档；
- `tests/` 测试；
- `requirements.txt`；
- `start_ophiuchus.bat`；
- README、版本号和科学边界说明。

默认不要共享：

- `.env`：可能包含个人 API Key 和本机程序路径；
- `data/ophi_app_state.json`：包含上次使用的本机绝对路径；
- `results/`、`exports/` 和临时分析：可能包含未公开数据；
- 私有 CIF、实验原始数据和受许可限制的数据库内容；
- 不准备共同维护的本地 SQLite 结构库。

如果确实要共享结构库，应先确认其中每个结构的来源和许可，并说明库版本。接收者首次启动后应重新设置 XRD、候选目录、结构库、VESTA/RIETAN 和 API 配置。

## 17. 故障排查

### 软件双击后没有窗口

在 Anaconda Prompt 中运行 `python -m ophiuchus app`，根据终端错误确认 Python 环境和依赖。不要只看批处理窗口一闪而过。

### 提示缺少 pymatgen 或 CIF 模拟失败

激活 `ophi` 环境，重新执行 `python -m pip install -r requirements.txt`。随后用一个已知 CIF 进行验证，不要在后端失败时继续相信模拟强度。

### Materials Project 全部采集失败

依次检查网络、API Key、`mp-api` 版本、元素范围和 Materials Project 服务状态。先测试连接，再使用保守模式采集少量体系。

### 结构库分析很慢

确认只选择了样品主元素，关闭明显无关结构，提前构建 XRD 缓存，并先用较小结构范围验证流程。最终展示候选可能需要 VESTA/RIETAN 重新模拟，速度会慢于读取缓存。

### 目标相突然不对

检查是否更换了实验谱却没有更新主元素和目标相。点击“刷新目标相”，重新明确选择；必要时重新导入目标相 CIF。

### 模拟峰位或峰强可疑

先追溯实际使用的 CIF、波长和计算后端，再与同一 CIF、同一辐射源设置下的 VESTA/RIETAN 结果比较。峰位一致但相对强度不同，可能来自织构、占位、热振动、展宽或仪器模型；峰位明显不一致时，不应继续用该模拟结果判断物相。

### 无法打开操作手册

确认 `docs/Ophiuchus_操作手册.md` 与软件目录一起存在，并为 `.md` 文件设置可用的默认打开程序。
